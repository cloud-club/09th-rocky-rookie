# 포트 매핑·볼륨 연결·기본 네트워크 설정

## 포트 매핑
- 호스트 OS의 커널 엔진이 들어오는 패킷을 가로채서 컨테이너의 가상 IP와 포트로 전달
```
[ External Client ]
       |
       | (Request to Host IP : 8080)
       v
+--------------------------------------------------------------+
| Host OS (Network Stack)                                      |
|                                                              |
|   [ NIC (eth0) ] <--- IP: 192.168.1.10                       |
|         |                                                    |
|         v                                                    |
|   [ Netfilter / IPTables (DNAT Rule) ]  <-- (포트 매핑 설정 지점)
|         |                                                    |
|         +----------( Translation: 8080 -> 10.88.0.2:80 )-----+
|         |                                                    |
|         v                                                    |
|   [ Virtual Bridge (cni-podman0) ]                           |
|         |                                                    |
|         +-----------------------+                            |
|                                 |                            |
|         (Virtual Cable)         | (Virtual Cable)            |
+---------|-----------------------|----------------------------+
          | veth-host             | veth-container
          v                       v
    +---------------------------------------+
    | Container (Network Namespace)         |
    |                                       |
    |  [ Virtual NIC (eth0) ]               |
    |    IP: 10.88.0.2                      |
    |         |                             |
    |         v                             |
    |  [ Application (Nginx) ] : Port 80     |
    +---------------------------------------+
```

### 포트 매핑 설정 위치
- 호스트 OS의 커널 영역에서 설정  
  - `podman run -p 8080:80` 명령을 실행하면, Podman은 호스트 커널의 IPTables(또는 nftables) 규칙에 DNAT(Destination NAT) 설정을 자동으로 추가 : 호스트의 8080번 포트로 들어오는 TCP 패킷의 목적지 주소를 컨테이너의 가상 IP(10.88.0.2)와 포트(80)로 변환하기

### 데이터 흐름
1. 패킷 유입 : 호스트의 무리 NIC로 패킷이 들어옴
2. 가로채기 : 커널이 패킷을 분석하여 미리 설정된 DNAT 규칙 발견
3. 주소 변환 : 목적지 IP를 컨테이너의 사설 IP로, 포트를 컨테이너 내부 포트로 변경
4. 라우팅 : 변환된 패킷을 가상 브리지로 전송
5. 도달 : 브리지는 연결된 veth 파이프를 통해 컨테이너 내부 애플리케이션으로 패킷 전달


## 볼륨 연결
### 설정 위치
- 컨테이너 생성 시 리눅스 커널의 VFS(Virtual File System) 계층에 설정  
  - 컨테이너만의 격리된 파일 시스템(네임스페이스) 구축 : 특정 경로에 대해 호스트의 실제 디렉토리를 바인드 마운트하도록 커널에 지시 

### 기존 호스트와의 연결
- 커널 수준에서 동일한 데이터 블록(Inode)을 가리키도록 연결하는 방식

### RHEL/Podman 설정 : :Z 옵션
- 호스트와 컨테이너가 경로를 연결하더라도, RHEL의 SELinux가 차단할 수 있음  
    -> `-v /host:/cont:Z` 옵션을 붙이면 마운트 직전에 해당 호스트 디렉토리에 `container_file_t`라는 보안 라벨 부착  
    -> 커널이 라벨 확인 후 컨테이너 접근 승인 


## 기본 네트워크 설정
- podman은 실행 CNI(Container Network Interfacre) 표준을 사용하여 가상 브리지 생성 및 사설 IP 할당 

### DNS 관리
- 컨테이너 내부의 `/etc/resolv.conf`를 관리하여 컨테이너끼리 이름을 통해 통신할 수 있도록 설정

### 주요 네트워크 모드
- **Bridge (기본값)** : 호스트와 분리된 가상 네트워크를 사용하며, 포트 매핑을 통해 통신
- **Host** : 호스트의 네트워크 스택을 그대로 공유 (보안 격리)
- **None** : 네트워크 연결 차단
