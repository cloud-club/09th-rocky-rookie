# 1. 리소스별 점검 도구

## CPU


### `lscpu`

```bash
lscpu
```

CPU 개수, 코어 수, 스레드 수, 아키텍처 같은 하드웨어 정보 확인

확인할 수 있는 항목:

- CPU 개수
- 코어 수
- 스레드 수
- 소켓 수
- 아키텍처
- 가상화 지원 여부

### `perf`

***설치 필요**

```bash
perf stat ls
perf top
```

`perf`는 CPU 성능 분석과 이벤트 측정을 위한 리눅스 성능 분석 도구이다.

**CPU가 어떤 코드 경로, 어떤 함수, 어떤 이벤트에서 시간을 쓰는지**를 분석할 때 사용한다.

대표 활용 예시는 다음과 같다.

- `perf stat 명령어`
    
    → 명령 실행 동안 CPU cycles, instructions, cache miss 같은 성능 이벤트 측정
        
- `perf top`
    
    → CPU를 많이 쓰는 함수/심볼을 실시간으로 확인
        
- `perf record`, `perf report`
    
    → 성능 샘플을 수집하고 사후 분석
    

## 메모리



### `free -h`

```bash
free -h
```

전체 메모리, 사용 메모리, 사용 가능 메모리, swap 사용량을 사람이 읽기 쉬운 단위로 보여준다.

메모리 점검 시에는 단순히 `used`만 보기보다 다음을 함께 본다.

- `available` 메모리가 충분한지
- swap 사용이 발생하고 있는지

### 커널 로그

```bash
journalctl -k --since "10 min ago"
sudo dmesg | tail
```

OOM(Out Of Memory) 같은 메모리 문제는 커널 로그에 남을 수 있다.

예를 들어, 다음 여부를 확인할 수 있다.

- OOM killer 동작 여부
- 커널이 특정 프로세스를 강제 종료했는지
- 메모리 관련 경고가 있었는지

## 디스크


디스크는 **용량 문제**와 **I/O 성능 문제**를 구분해서 봐야 한다.

### 용량 확인

### `df -h`

```bash
df -h
```

파일시스템별 사용량과 남은 용량을 확인한다.

주로 다음을 본다.

- 어느 파일시스템이 가득 찼는지
- 여유 공간이 충분한지
- 서비스 장애 원인이 용량 부족인지

### `lsblk`

```bash
lsblk
```

디스크와 파티션, 마운트 구조를 트리 형태로 보여준다.

### I/O 성능 확인

### `iostat`

***설치 필요**

```bash
iostat -x 1 5
```

디스크가 느린지 판단할 때 쓰는 대표적인 도구이다.

디스크 장치별 부하, 전송률, 대기시간을 확인한다.

주로 다음 항목을 본다.

- 읽기/쓰기 처리량
- `await` : 평균 I/O 대기 시간
- `%util` : 디스크 사용률

예를 들어:

- `await`가 높다 → 요청 처리 지연 가능성
- `%util`이 높다 → 디스크 장치 포화 가능성

## 프로세스

### `ps`

```bash
ps aux --sort=-%cpu | head
ps aux --sort=-%mem | head
```

CPU를 많이 쓰는 프로세스와 메모리를 많이 쓰는 프로세스를 빠르게 확인한다.

점검 포인트:

- CPU를 많이 쓰는 프로세스
- 메모리를 많이 쓰는 프로세스
- 비정상적으로 중복 실행된 프로세스
- 재시작이 반복되는 프로세스
- 좀비 프로세스 존재 여부

### `pidstat`

***설치 필요**

```bash
# 1초 간격으로 5번 샘플링
pidstat 1 5

# 특정 PID 추적
pidstat -p 1234 1 5
```

프로세스별 CPU, 메모리, I/O 사용 변화를 주기적으로 확인할 수 있다.

`ps`가 현재 시점의 정렬된 목록이라면, `pidstat`은 **시간에 따라 변하는 프로세스 부하**를 보는 데 더 적합하다.

# 2. 시스템 전체 모니터링 도구

## `top`

```bash
top
```

실시간으로 시스템 전체와 프로세스별 CPU/메모리 사용량을 본다.

확인 포인트:

- CPU를 많이 먹는 프로세스
- 메모리를 많이 먹는 프로세스
- load average
- 시스템 전체 CPU 상태
- 메모리 / swap 상태

## `vmstat 1 5`

```bash
vmstat 1 5
```

1초 간격으로 5번 샘플링해서, 다음 정보를 한 번에 본다.

- 프로세스 상태
- 메모리
- paging
- block I/O
- CPU 상태

특히 다음 상황을 함께 볼 수 있어 유용하다.

- CPU 사용률이 높은지
- `wa`(I/O wait)가 높은지
- swap in/out이 발생하는지
- block I/O가 몰리는지

## `sar`

***설치 필요**

```bash
sar -u 1 5
sar -n DEV 1 5
```

`sar`는 CPU, 메모리, 네트워크 등 **시스템 자원의 사용 추이**를 주기적으로 수집·조회하는 도구이다.

대표 활용:

- `sar -u` : CPU 사용률 변화
    
    
- `sar -n DEV` : 네트워크 인터페이스별 송수신량
- `sar -r` : 메모리 사용 추이

# 3. 활용하기



## CPU 병목이 의심될 때

- `top`
- `vmstat`
- `lscpu`
- `perf`

→ CPU 사용량, load average, I/O wait, CPU 구성, 함수 단위 병목 확인하기

## 메모리 부족이 의심될 때

- `free -h`
- `vmstat`
- `journalctl -k`
- `dmesg`

→ available 메모리, swap, paging, OOM 흔적 확인하기

## 디스크 병목이 의심될 때

- `df -h`
- `lsblk`
- `iostat -x`
- `vmstat`

→ 용량 문제와 I/O 성능 문제를 구분하기

## 문제 프로세스를 찾고 싶을 때

- `top`
- `ps aux --sort=-%cpu`
- `ps aux --sort=-%mem`
- `pidstat`

→ 자원을 많이 쓰는 프로세스와 시간에 따른 변화 추적하기
