# 1. Bash Shell의 기본 구조와 이해
## Bash Shell이란?
- 사용자가 입력한 명령어를 해석해서 OS에 전달하는 프로그램
- 사용자와 리눅스 커널 사이의 중간 관리자 역할을 한다.

## Bash Shell의 기본 동작 흐름
1. 사용자가 프롬프트에서 명령 입력
2. Bash가 입력 내용을 읽음
3. 명령어, 옵션, 인자를 구분해서 해석
4. 내장 명령인지 외부 프로그램인지 확인
5. 실행
6. 결과를 화면에 출력

### 기본 형태
user@host:~$(프롬프트) ls(명령어) -l(옵션) /home(인자)

- 프롬프트(user@host:~$): 명령 입력을 기다리는 표시
- 명령어(ls): Bash가 실행할 작업
- 옵션(-l): 명령어의 동작 방식을 바꾸는 요소
- 인자(/home): 명령어가 처리할 대상

### 명령어의 2가지 Type
1. 내장 명령어
   - 쉘에 내장되어 있는 명령어. 쉘이 명령어를 이해하고 수행.
   - ex) cd, echo, pwd
2. 외부 명령어
   - /bin 디렉토리 안에 파일의 형태로 존재하며 사용하는 명령어. PATH 환경변수 내의 디렉토리들을 검색하여 해당 파일(/bin 등)을 찾은 뒤, fork()를 통해 새로운 서브프로세스를 생성하여 명령어를 실행한다.
