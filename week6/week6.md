# 1. Bash Shell의 기본 구조와 이해
## Bash Shell이란?
- 사용자가 입력한 명령어를 해석해서 OS에 전달하는 프로그램
- 사용자와 리눅스 커널 사이의 중간 관리자 역할을 한다.

## Bash Shell의 기본 동작 흐름
┌──────────────┐
│    사용자     │
│ (키보드 입력) │
└──────┬───────┘
       │ 명령 입력
       ▼
┌──────────────┐
│  Bash 셸     │
│  (명령 해석기) │
└──────┬───────┘
       │
       ├─ 1) 명령어 해석
       ├─ 2) 변수/기호 확장
       ├─ 3) 실행 대상 확인
       ▼
┌──────────────┐
│ 실행 대상 결정 │
├──────────────┤
│ 내장 명령어    │
│ 또는 외부 프로그램 │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 커널 / 시스템  │
│ 자원에 실행 요청 │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 실행 결과 출력 │
│ 화면 / 파일 /  │
│ 다른 명령어로 전달 │
└──────────────┘
1. 사용자가 프롬프트에서 명령 입력
2. Bash가 입력 내용을 읽음
3. 명령어, 옵션, 인자를 구분해서 해석
4. 내장 명령인지 외부 프로그램인지 확인
5. 실행
6. 결과를 화면에 출력

### 기본 형태
user@host:~$(프롬프트) ls(명령어) -l(옵션) /home(인자)

- 프롬프트(user@host:~$): 명령 입력을 기다리는 표시
- 명령어(ls): Bash가 실행할 작업
- 옵션(-l): 명령어의 동작 방식을 바꾸는 요소
- 인자(/home): 명령어가 처리할 대상

### 명령어의 2가지 Type
1. 내장 명령어
   - 쉘에 내장되어 있는 명령어. 쉘이 명령어를 이해하고 수행.
   - ex) cd, echo, pwd
2. 외부 명령어
   - /bin 디렉토리 안에 파일의 형태로 존재하며 사용하는 명령어. PATH 환경변수 내의 디렉토리들을 검색하여 해당 파일(/bin 등)을 찾은 뒤, fork()를 통해 새로운 서브프로세스를 생성하여 명령어를 실행한다.

# 2. 표준 입력·출력·리다이렉션과 파이프 처리
             ┌─────────────┐
             │   키보드     │
             │  (표준입력)   │
             └──────┬──────┘
                    │
                    ▼
              ┌──────────┐
              │  명령어   │
              │ (process)│
              └────┬─────┘
         ┌─────────┼─────────┐
         ▼         ▼         ▼
   ┌─────────┐ ┌─────────┐ ┌─────────┐
   │ stdout  │ │ stderr  │ │ 다른 명령 │
   │정상 출력 │ │오류 출력 │ │  로 전달  │
   └─────────┘ └─────────┘ └─────────┘
         │
         ▼
      화면/파일
Bash에서 명령어는 단순히 실행만 되는 게 아니라 입력받고 → 처리하고 → 출력하는 흐름으로 동작한다.
이 과정에서의 핵심이 되는 것은 아래와 같다. 

### 1.표준 입력(stdin)
- 프로그램이 입력을 받는 기본 통로. 또한 파일을 표준 입력으로 사용할 수 있다.
  sort < names.txt

### 2. 표준 출력(stdout)
- 프로그램이 정상 결과를 내보내는 기본 통로

### 3. 표준 에러(stderr)
- 오류 메시지를 내보내는 기본 통로
이것도 기본적으로 화면에 보이지만, stdout과는 별도의 통로이다.
